## 1. ダッシュボードの正しい使い方

### 1.1 最適な閲覧ワークフロー

プロスキャルパーはダッシュボードの全パネルを均等に見ない。**情報には優先順位がある。** 以下の順番で確認すること。

```
Step 1: 複合シグナル（ページ上部ヒーロー）
  → LONG / SHORT / WAIT を確認。これが全体の方向性
  → 5段階の強度バー（Absolute SHORT〜Absolute LONG）で確信度を把握

Step 2: タイムフレーム一致確認（5枚のTFカード）
  → 5m / 15m / 1h / 4h / 1d が同方向なら高確信
  → コンフリクトパネルで不一致を確認
  → 【重要】上位足（4h, 1d）と下位足（5m, 15m）が逆方向なら見送り

Step 3: エントリータイミング（1分足解析セクション）
  → GOなら即座にエントリー準備
  → WAIT_ENTRYなら数分待機
  → AGAINSTなら絶対に入らない

Step 4: インジケータ詳細カード
  → RSI, Trend, FR, L/S, VWAP, BB, Delta を個別確認
  → 1つでも「強い逆シグナル」があればサイズを縮小

Step 5: 価格レンジ予測セクション
  → レジスタンス/サポートレベルでSL/TPを決定
  → TP1/TP2/TP3の目安時間を確認
```

### 1.2 更新速度の使い分け

| 状況 | 推奨速度 | 理由 |
|------|----------|------|
| ポジション保有中 | **1秒** | 価格変動をリアルタイムで追跡。逃げ遅れ防止 |
| セットアップ探索中 | **3秒** | エントリーポイントを探しつつAPI負荷を抑制 |
| モニタリング（待機） | **30秒** | 大局観の確認のみ。APIレート制限回避 |

### 1.3 重要な注意事項

#### ハードコードされたデータに注意

以下のデータは**ライブAPIではなく手動更新**のため、表示が古い可能性がある：

- **BTC ETFフロー**（「直近5日・週次手動更新」と記載）: 機関資金の流入/流出。週1回手動更新のため、最新値はFarside Investorsで別途確認すること
- **市場センチメント総合**（取引所BTC残高、P/C比率、Googleトレンド、マクロ）: これらは**すべて固定値**。リアルタイムデータではない
- **重要ファンダメンタルズ分析**: 「AI分析」と表示されるが、実際はハードコードされたイベントリスト（NFP, CPI, FOMC等）

**これらはスキャルピングの直接判断には使わない。** 大局観の参考情報として「こういう環境だ」と認識するのみ。

#### コインスイッチャーの挙動

BTC / ETH / XRP / SOL の切り替え時：
- 全データがリセットされ、新しいコインのデータを再取得する
- スコアヒストリーもリセットされる
- ETFフローは BTC と ETH のみ表示される

### 1.4 各パネルの役割一覧

|----------|------|-------------------|

## 2. スコアリングエンジン解剖

### 2.1 スコア計算の全体像

ダッシュボードの核心は `computeTF` 関数。各タイムフレーム（5m/15m/1h/4h/1d）ごとに以下のプロセスでスコアを算出する：

```
1. 各インジケータを計算
2. 閾値に基づいてbull（強気）ポイント / bear（弱気）ポイントを加算
3. スコア = bull / (bull + bear) × 100
4. スコア ≥ 65 → LONG、≤ 35 → SHORT、36〜64 → WAIT
```

### 2.2 インジケータ別の詳細

#### RSI（相対力指数）

| 設定 | 値 |
|------|-----|
| 期間 | 5m/15m = **9本**（高速反応）、1h/4h/1d = **14本**（標準） |
| 重み | 最大 **2ポイント** |

| RSI値 | 判定 | スコアへの影響 |
|--------|------|---------------|
| > 75 | 買われすぎ（bearish） | bear +2 |
| 60〜75 | 強気モメンタム | bull +1 |
| 40〜60 | 中立 | 影響なし |
| 25〜40 | 弱気モメンタム | bear +1 |
| < 25 | 売られすぎ（bullish） | bull +2 |

**プロの視点**: RSI 9は5m足では有効だが、1h以上では14が標準。現在の実装は適切。ただし閾値が静的なのが問題（後述）。

#### SMAトレンド（移動平均線）

SMA20とSMA50のクロスオーバーを判定。タイムフレームごとに `trW`（トレンド重み）が異なる。

| タイムフレーム | トレンド重み (trW) |
|---------------|-------------------|
| 5m | 0.4（弱い影響） |
| 15m | 0.55 |
| 1h | 0.7 |
| 4h | 1.0（フル影響） |
| 1d | 1.0 |

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| 価格 > SMA20 > SMA50 | 強い上昇トレンド | bull + 2 × trW |
| 価格 > SMA20 | やや上昇 | bull + 1 × trW |
| 価格 < SMA20 < SMA50 | 強い下降トレンド | bear + 2 × trW |
| 価格 < SMA20 | やや下降 | bear + 1 × trW |
| その他 | 横ばい | 影響なし |

**プロの視点**: SMA20/50は定番だが、スキャルではEMA（指数移動平均）の方が反応が速い。EMA9/21の組み合わせがスキャルでは一般的。

#### Funding Rate（資金調達率）

タイムフレームごとに `frW`（FR重み）が異なる。

| タイムフレーム | FR重み (frW) |
|---------------|-------------|
| 5m | 0.25（ほぼ無視） |
| 15m | 0.45 |
| 1h | **0.9（最重視）** |
| 4h | 0.7 |
| 1d | 0.35 |

計算: `実効FR = S.fr × frW`

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| 実効FR > 0.025 | ロング過多→逆張りショート | bear + 2 × frW |
| 実効FR < -0.015 | ショート過多→逆張りロング | bull + 2 × frW |
| その他 | FR中立 | 影響なし |

**プロの視点**: FRは8時間ごとに精算されるため、5m足への影響は限定的。frW=0.25は妥当。ただし、FR精算直前（00:00/08:00/16:00 UTC の30分前）はFR狙いのポジション調整が起きるため、この時間帯のシグナルは信頼度が下がる。

#### L/S比率（ロング/ショート比率）

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| ロング比率 > 62% | ロング過多（bearish） | bear +1.2 |
| ロング比率 < 38% | ショート過多（bullish） | bull +1.2 |
| 38〜62% | 均衡 | 影響なし |

**プロの視点**: L/S比率は全体のアカウントベースの比率であり、大口の実際のポジションサイズは反映されない。参考程度にとどめること。

#### Fear & Greed Index（恐怖・強欲指数）

| タイムフレーム | F&G重み (fgW) |
|---------------|-------------|
| 5m / 15m | 0.1（ほぼ無視） |
| 4h | 0.5 |
| 1d | **1.0** |

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| F&G ≥ 80 | 極度の強欲（bearish） | bear + 1.5 × fgW |
| F&G ≤ 25 | 極度の恐怖（bullish） | bull + 1.5 × fgW |

**プロの視点**: 日次更新のためスキャルでは事実上無意味。5m/15mでfgW=0.1は正しい設計判断。

#### VWAP（出来高加重平均価格）

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| 価格 > VWAP × 1.002 | VWAP上方 | bull +1.5 |
| 価格 < VWAP × 0.998 | VWAP下方 | bear +1.5 |
| VWAP付近（±0.2%以内） | 中立 | 影響なし |

**プロの視点（重要な限界）**: 現在のVWAP計算は**セッション区切りがない**。通常、VWAPは毎日UTC 00:00でリセットすべき。リセットなしでは全klineの累積平均となり、時間が経つほど現在価格に近づいて意味を失う。

#### ボリンジャーバンド

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| 価格 > 上バンド | 過熱（bearish） | bear +1.5 |
| 価格 < 下バンド | 売られすぎ（bullish） | bull +1.5 |
| バンド幅 < 2.0% | スクイーズ（ブレイクアウト準備） | 影響なし（警告のみ） |
| その他 | バンド内推移 | 影響なし |

**プロの視点**: BBバンド外へのブレイクは「反転」ではなく「モメンタム継続」のサインになることも多い。特にBBスクイーズ後のブレイクアウトは強いトレンドの始まりであり、逆張りは危険。

#### Delta（売買圧力）

5m/15mのみ適用（`fast: true`のTFのみ）。

| 条件 | 判定 | スコアへの影響 |
|------|------|---------------|
| Delta > 0.3 | 買い圧優位 | bull +1.2 |
| Delta < -0.3 | 売り圧優位 | bear +1.2 |
| -0.3〜0.3 | 拮抗 | 影響なし |

**プロの視点（重大な限界）**: 現在のDelta計算は `(close - open) / (high - low)` というローソク足の実体比率を使った**近似値**にすぎない。Binance kline APIは各足の `takerBuyBaseAssetVolume`（フィールド`[9]`）を返しており、これを使えば**実際のテイカー買い量と売り量の差**を計算できる。現在この値は取得されているが**使われていない**。

### 2.3 複合スコアの計算

5つのタイムフレームのスコアを加重平均する：

| タイムフレーム | 重み | 理由 |
|---------------|------|------|
| 5m | 1.0 | スキャルの主力だが、ノイズが多い |
| 15m | 1.5 | エントリー確認TF |
| 1h | 2.0 | 中期トレンドの把握 |
| 4h | **2.5**（最大） | 大局的な方向性。最も信頼度が高い |
| 1d | 2.0 | 日足レベルのバイアス |

```
複合スコア = (5m×1 + 15m×1.5 + 1h×2 + 4h×2.5 + 1d×2) / 9.0
```

**プロの視点**: 4hに最大重みを置くのは正しい。スキャルでも「大きな流れに逆らわない」のが鉄則。ただし、4hと5mが逆方向の場合、複合スコアが中立（50付近）に落ち着いてしまい「WAIT」になりがち。これは**正しい挙動**だが、実際にはそのような状況でも4h方向への5m反転タイミングで入れるチャンスがある。

### 2.4 5段階シグナル判定

| 複合スコア | 判定 | 表示 |
|-----------|------|------|
| 0〜20 | Absolute SHORT | 赤ダブル点滅 |
| 21〜39 | SHORT寄り | オレンジ |
| 40〜60 | WAIT | グレー |
| 61〜79 | LONG寄り | 青 |
| 80〜100 | Absolute LONG | 緑ダブル点滅 |

### 2.5 既知の限界と対処法

#### 限界1: 静的閾値

全インジケータの閾値（RSI 25/75、FR 0.025等）は固定値。市場のボラティリティが変わっても閾値は変わらない。

**対処法**: 強いトレンド中はRSIが60〜80で推移し続けることがある。「RSI 75で売り」は逆張りの罠になりうる。上位足のトレンド方向を必ず確認し、トレンド方向への60超えは「まだ行ける」と解釈すること。

#### 限界2: スコア計算式の特性

`bull/(bull+bear)×100` という計算式は、少数のインジケータしか発火しない場合に極端な値を出す。例えば、RSIだけがbear+2で他が全て中立の場合、スコアは0（Absolute SHORT）になるが、これは1つのインジケータだけの判断であり信頼度は低い。

**対処法**: インジケータ詳細カードを見て、**いくつのインジケータが発火しているか**を確認すること。3つ以上が同方向なら信頼度が高い。

#### 限界3: ヒストリカルスコアの不正確さ

スコアヒストリーチャートは過去のスコア推移を表示するが、**FR（Funding Rate）とL/S比率は現在の値を過去に遡及適用**している。つまり、過去のスコアは「もし今のFR/L/Sが当時も同じだったら」という仮定のスコアであり、当時の実際のスコアとは異なる。

**対処法**: ヒストリーチャートはスコアのトレンド（上昇中/下降中）の参考にとどめ、過去の絶対値を信用しないこと。

## 3. エントリータイミング完全解説

### 3.1 エントリータイミングシステムの仕組み

ダッシュボードの「1分足エントリータイミング」セクションは、以下の2段階で判断する：

#### ステップ1: 上位足の方向投票

| タイムフレーム | 重み | LONG/SHORT |
|---------------|------|-----------|
| 1H | 1 | そのTFのシグナルをそのまま使用 |
| 4H | **2** | 最重視 |
| 1D | 1 | 日足のバイアス |

例: 1H=LONG(+1)、4H=LONG(+2)、1D=SHORT(-1) → LONG票3 vs SHORT票1 → **LONG方向**

#### ステップ2: 1分足スコアリング

ベーススコア50点からスタートし、上位足の方向に「乗っているか」を判定する。

**LONG方向の場合:**

| 条件 | 加算/減算 |
|------|----------|
| RSI < 40（押し目） | +15 |
| RSI < 50 | +8 |
| RSI > 70（過熱） | -15 |
| 価格 > SMA10 | +10 |
| 価格 > SMA20 | +8 |
| モメンタム > 0（5本前比上昇） | +10 |
| モメンタム < 0 | -8 |
| 出来高 > 平均 × 1.3 | +7 |

**SHORT方向の場合は逆のロジック。**

#### ステップ3: 判定

| スコア | 判定 | アクション |
|--------|------|-----------|
| ≥ 70 | **GO** | エントリー好機 |
| 55〜69 | **WAIT_ENTRY** | もう少し待機 |
| 35〜54 | **WAIT** | タイミング待ち |
| < 35 | **AGAINST** | 逆行中。絶対に入らない |

### 3.2 エントリータイミングの実践的な使い方

#### GOが出たとき

1. **1分足の現在のローソク足が確定するまで待つ**（ヒゲの先端でエントリーすると不利約定のリスク）
2. ローソク足が確定したら、次の足の始値でエントリー
3. SLは直近のスイング安値（LONG）/ 高値（SHORT）の少し外側に設定

#### WAIT_ENTRYが出たとき

1. スコアが70に近づくまで待機（1〜3分程度）
2. 1分足チャートで価格がSMA10に戻るのを待つ
3. SMA10でサポートされたら（LONG）/ 抵抗されたら（SHORT）エントリー

#### AGAINSTが出たとき

**絶対にエントリーしない。** 上位足の方向と1分足が逆行している。これは「もう少し待てば良いエントリーポイントが来る」サインであり、焦って入る場面ではない。

### 3.3 このシステムが見逃していること

現在のエントリータイミングには以下の分析が**含まれていない**：

1. **スプレッド拡大の検知**: スプレッドが広がっているとき（流動性が薄い）はエントリーを控えるべきだが、この判定がない
2. **1分足と5分足のモメンタム乖離**: 1分足のみを見ており、5分足のモメンタムとの整合性を確認していない
3. **ローソク足パターン認識**: ハンマー、シューティングスター、包み足などのリバーサルパターンを検出していない
4. **板の厚み（DOM）**: 上位足方向のサポート/レジスタンスに大口注文があるかを確認できない

**対処法**: GOシグナルが出ても、以下を自分の目で確認してからエントリーすること：
- TradingViewチャートで1分足のローソク足形状を確認
- 明確なリバーサルパターン（ハンマー等）があればエントリーの確信度アップ
- 直前に大きなヒゲ（上下ともに長い足）が出ていたらボラ急変の兆候、見送り

## 4. ダッシュボードに足りないもの（プロ視点）

このセクションは、プロスキャルパーが日常的に使っているが、このダッシュボードには**実装されていないデータ・分析手法**をまとめる。これらを理解し、外部ツールで補完することで勝率は大幅に上がる。

### 4.1 オーダーフロー分析（Order Flow）

#### DOM（板情報）とは

DOM（Depth of Market）は、取引所の注文板に並んでいる指値注文の一覧。価格の各レベルにどれだけの買い/売り注文が待っているかを視覚化する。

```
[売り板]
$87,500  |  12.5 BTC  ←← 大口の売り壁
$87,450  |   3.2 BTC
$87,400  |   1.8 BTC
--- 現在価格 $87,350 ---
$87,300  |   2.1 BTC
$87,250  |   4.5 BTC
$87,200  |  15.3 BTC  ←← 大口の買い壁（サポート）
[買い板]
```

#### 板情報から読み取れること

- **買い壁（大口の買い指値）**: そのレベルではサポートが強い。価格が下がりにくい
- **売り壁（大口の売り指値）**: そのレベルではレジスタンスが強い。価格が上がりにくい
- **壁が急に消える**: 大口トレーダーがキャンセルした → その方向への動きが加速する可能性
- **スプーフィング**: 大口注文を出しては即キャンセルする行為。方向を偽装する操作のため、壁を見て反射的にエントリーすると罠にかかる

#### なぜダッシュボードにないのか

板情報はWebSocket APIでリアルタイム取得する必要があり、REST APIでは十分な情報が得られない。また、板は常に変動しており「スナップショット」では意味がない。

#### 補完方法

- **Bookmap**: 板の厚みとヒートマップを視覚化する最強ツール
- **TensorCharts**: 板情報とCVDを統合表示
- Binanceの取引画面の板情報でも最低限の確認は可能

### 4.2 CVD — Cumulative Volume Delta（出来高デルタ累積）

#### CVDとは

CVDは「テイカー（成行注文）の買い量 - 売り量」の累積値。

```
CVD = Σ (テイカー買い量 - テイカー売り量)
```

- CVD上昇 = 成行買いが優勢 = 積極的な買い手がいる
- CVD下降 = 成行売りが優勢 = 積極的な売り手がいる

#### ダッシュボードの「Delta」との違い

ダッシュボードのDeltaは `(close - open) / (high - low)` という**ローソク足の形状から推定した近似値**。実際のテイカー買い/売り量は使っていない。

Binance kline APIは各足のデータの中に `takerBuyBaseAssetVolume`（フィールドインデックス `[9]`）を返しており、これは**実際にテイカー（成行注文）で買われたBTCの量**。このデータは現在取得されているが、コード上で使われていない。

**実際のCVD計算:**
```
各足のDelta = takerBuyVolume - (totalVolume - takerBuyVolume)
CVD = 各足のDeltaの累積和
```

#### CVDダイバージェンスの重要性

| パターン | 意味 | アクション |
|----------|------|-----------|
| 価格上昇 + CVD上昇 | 健全な上昇。買い手が積極的 | トレンドフォロー（ロング） |
| 価格上昇 + CVD下降 | **弱気ダイバージェンス**。価格は上がっているが実際は売り優勢 | ロング見送り / ショート準備 |
| 価格下降 + CVD下降 | 健全な下降。売り手が積極的 | トレンドフォロー（ショート） |
| 価格下降 + CVD上昇 | **強気ダイバージェンス**。価格は下がっているが実際は買い集め（アキュムレーション） | ショート見送り / ロング準備 |

**これはプロスキャルパーが最も重視するシグナルの1つ。** ローソク足のDelta近似では検出精度が低い。

### 4.3 Volume Profile（出来高プロファイル）

#### 概念

Volume Profileは、**各価格レベルで取引された出来高**を横棒グラフで表示する分析手法。

```
$88,000 | ████
$87,800 | ██████████████████  ← VAH (Value Area High)
$87,600 | ████████████████████████████  ← POC (最多出来高価格)
$87,400 | ██████████████████  ← VAL (Value Area Low)
$87,200 | ██████
$87,000 | ███
```

#### 重要な概念

| 用語 | 説明 | トレードでの使い方 |
|------|------|-------------------|
| **POC** (Point of Control) | 最も出来高が多い価格レベル | **価格の磁石**。POCに向かって価格が引き寄せられる傾向 |
| **VAH** (Value Area High) | 出来高の70%が集中する範囲の上限 | VAH上方ブレイクは強気。VAHでの反落はショートチャンス |
| **VAL** (Value Area Low) | 出来高の70%が集中する範囲の下限 | VAL下方ブレイクは弱気。VALでの反発はロングチャンス |
| **Low Volume Node** | 出来高が少ない価格帯 | 価格がこの帯を通過すると**高速で動く**（抵抗がない） |
| **High Volume Node** | 出来高が多い価格帯 | 価格がこの帯に入ると**動きが鈍化**する |

#### スキャルでの活用

1. **POCバウンス**: 価格がPOCに戻ったとき、バウンスを狙う（POCは機関トレーダーの「公正価格」とみなされる）
2. **VAH/VAL ブレイク**: Value Areaの外に出た場合、そのまま加速する可能性が高い
3. **Low Volume Nodeの高速通過**: 出来高が薄い帯を抜けると、次のHigh Volume Nodeまで一気に動く

#### 補完方法

ダッシュボードにはVolume Profile機能がないが、TradingViewの「Volume Profile」インジケータ（固定範囲 or セッション）を追加すれば同等の分析が可能。

### 4.4 清算ヒートマップ（Liquidation Heatmap）

#### ダッシュボードの限界

現在の価格レンジ予測セクションでは、清算レベルを `price × 1.025`（2.5%上下）と単純に推定している。実際の清算クラスタはこの推定とは全く異なる分布をしている。

#### 清算ヒートマップとは

取引所のオープンポジションから推定される**強制清算（ロスカット）が集中する価格帯**を視覚化したもの。

```
$89,000 | ░░░░░░░░  (少数の清算)
$88,500 | ████████████████████  ← 大量のSHORT清算（黄色/白 = 高密度）
$88,000 | ██████
--- 現在 $87,500 ---
$87,000 | ████████
$86,500 | ████████████████████████████  ← 大量のLONG清算
$86,000 | ██████████
```

#### なぜ清算クラスタが重要なのか

- 清算は**キャンセルできない強制成行注文**
- 大量のロング清算が発生すると → 大量の成行売りが出る → 価格がさらに下がる → さらに清算が連鎖（**清算カスケード**）
- 逆に、清算クラスタに向かって価格が動くと**清算を刈り取る動き（liquidation hunt）**が起きやすい
- 清算クラスタは**価格の磁石**として機能する

#### スキャルでの活用

1. 現在価格の上下2〜5%以内に清算クラスタがあるか確認
2. 上位足のトレンドがクラスタ方向を向いているか確認
3. クラスタ方向にエントリーし、クラスタ直前でTP（清算が起きると一気に動くが、その後すぐ反転することが多い）

#### 補完方法

- **CoinGlass Liquidation Heatmap**: 最も信頼度の高い無料ツール
- 3H / 12H / 24H / 1W / 1M の清算データを確認可能

### 4.5 相関分析（Correlation Analysis）

BTCは単独で動いていない。以下の市場との相関を理解することで、BTC単体のチャートでは見えない「次の動き」が予測しやすくなる。

| 市場 | BTC との関係 | 確認方法 |
|------|------------|----------|
| **DXY**（ドルインデックス） | **逆相関**。DXY上昇→BTC下落圧力 | TradingView で DXY チャートを確認 |
| **S&P 500** (SPX) | **正相関**（リスクオン時）。SPX上昇→BTC追随 | 米株時間帯（22:30-05:00 JST）に特に連動 |
| **US 10Y 利回り** | **逆相関**。利回り上昇→リスク資産全体に逆風 | FOMC前後で特に重要 |
| **ETH/BTC 比率** | BTC dominance。比率下落→BTC独歩高（安全資産化） | アルトシーズンの判断に使用 |
| **USDT.D**（テザードミナンス） | **逆相関**。USDT.D上昇→仮想通貨全体から資金流出 | パニック売りの先行指標 |
| **Gold (XAU)** | 弱い正相関。ゴールド急騰→地政学リスク→BTC追随の可能性 | 有事のリスクオフ判断 |

#### スキャルでの実践的な使い方

1. **NY市場オープン前（22:00 JST）**: SPX先物の方向を確認。SPXが強ければBTCも上方向バイアス
2. **FOMC直後**: DXYと10Y利回りの反応を先に見る。BTCよりも先に動く
3. **週末明け月曜**: 週末のBTC動きがCMEギャップを作る。SPXの寄り付き方向も参考に

### 4.6 マーケットマイクロストラクチャ

#### スプレッド（Bid-Ask Spread）

- 通常のBTC先物スプレッドは $0.1〜$1.0 程度
- スプレッドが**急拡大**（$5以上）すると、マーケットメーカーが撤退している → **大きな動きの前兆**
- スプレッド拡大中のエントリーは不利約定（スリッページ）のリスクが高い

#### タイム＆セールス（約定履歴）

- **連続する大口約定**（1BTC以上の成行注文が連発）: 機関が動いている
- **大口買いの後に即座に大口売り**: ブロックトレードのヘッジ。方向感は中立
- **小口注文の連打**: リテール（個人）のパニック売買。逆張りのチャンス

## 5. セッション分析と時間帯戦略

### 5.1 3大セッションの特徴

BTCは24時間取引されるが、**時間帯によってボラティリティと流動性が全く異なる。** ダッシュボードはこの区別をしていないため、トレーダー自身が意識する必要がある。

| セッション | 時間（JST） | 時間（UTC） | 特徴 |
|-----------|------------|------------|------|
| **アジア** | 09:00〜17:00 | 00:00〜08:00 | 低ボラ、レンジ相場が多い。偽ブレイクアウトに注意 |
| **ロンドン** | 17:00〜01:00 | 08:00〜16:00 | ボラティリティが拡大し始める。アジアレンジのブレイクが起きやすい |
| **ニューヨーク** | 22:00〜07:00 | 13:00〜22:00 | **最高ボラ**。メジャーなトレンドが発生。経済指標発表あり |
| ロンドン-NY重複 | **22:00〜01:00** | 13:00〜16:00 | **ピークボラティリティ**。1日で最も大きな動きが出る時間帯 |
| オフピーク | 06:00〜09:00 | 21:00〜00:00 | NY終了〜アジア開始の端境期。流動性最低 |

### 5.2 セッション別スキャル戦略

#### アジアセッション（09:00〜17:00 JST）

```
推奨戦略: レンジスキャルピング
- アジアセッションの高値/安値を特定
- 範囲の上限でショート、下限でロング
- ブレイクアウトを追いかけない（偽ブレイクの確率が高い）
- 利確幅は小さめ（ATRの0.3倍程度）
- ダッシュボードのシグナルが極端な値を示しても鵜呑みにしない
```

**注意**: アジア時間帯にダッシュボードが「Absolute LONG」を出しても、ロンドンオープンで全く逆の動きが出ることは珍しくない。アジアのシグナルは信頼度30%減で解釈すること。

#### ロンドンセッション（17:00〜01:00 JST）

```
推奨戦略: アジアレンジブレイクアウト
- アジアセッションの高値/安値を確認
- 17:00〜18:00にブレイクが発生しやすい
- ブレイク後のリテスト（押し目/戻り）を待ってエントリー
- 利確はATRの0.6〜0.8倍
- ダッシュボードの1分足エントリータイミングが特に有効な時間帯
```

#### ニューヨークセッション（22:00〜07:00 JST）

```
推奨戦略: トレンドフォロー（順張り）
- 米株市場との相関が強まる時間帯
- 22:30 に経済指標（NFP, CPI等）がある場合は発表前30分にポジションクローズ
- 指標発表後の最初の5分足確定を待ってからエントリー
- ダッシュボードの複合シグナルが最も信頼できる時間帯
- 利確はATRの0.8〜1.2倍
```

### 5.3 経済指標カレンダーとの連動

#### 絶対に覚えるべき指標

| 指標 | 発表時間（JST） | BTC への影響 |
|------|----------------|-------------|
| **NFP（非農業部門雇用者数）** | 毎月第1金曜 22:30 | 強い雇用→ドル高→BTC下落圧力 |
| **CPI（消費者物価指数）** | 毎月中旬 22:30 | 高インフレ→利上げ懸念→BTC下落 |
| **FOMC（金利決定）** | 年8回 03:00〜04:00 | **最大インパクト**。声明文で上下$500以上動くことあり |
| **PPI（生産者物価指数）** | 毎月中旬 22:30 | CPI の先行指標 |
| **失業保険申請件数** | 毎週木曜 22:30 | インパクト中。予想との乖離が大きい場合のみ |

#### 指標発表時のルール

```
1. 発表 30分前: ポジションをクローズ or SLをタイトに
2. 発表直後: エントリーしない（スプレッド拡大、ヒゲが出やすい）
3. 発表後 5分: 最初の5分足が確定するまで待つ
4. 発表後 5〜15分: 方向が確定したらエントリー
5. 発表後 30分以降: ノイズが収束。通常のスキャルに戻る
```

### 5.4 FR精算タイミング

Funding Rateの精算は **08:00 / 16:00 / 00:00 UTC**（JST: 17:00 / 01:00 / 09:00）。

精算前30分（特に精算前15分）は：
- FR受け取り側が「精算まで持つ」ためにポジション維持
- FR支払い側が「精算前にクローズ」するため動きが出る
- 精算直後にポジション調整の反転が起きやすい

**スキャルへの示唆**: FR精算前後15分はノイズが増える。この時間帯のダッシュボードシグナルは割り引いて判断すること。

## 6. リスク管理の実践

### 6.1 ポジション計算機の使い方と限界

ダッシュボードのポジション計算機は4つのリスクティアを提供する：

| ティア | レバレッジ | 1トレード最大損失 | 推奨対象 |
|--------|-----------|-----------------|----------|
| 安全 | 2x | 資金の1% | 初心者・学習中 |
| 一般 | 3x | 資金の1.5% | 標準的なスキャルパー |
| 中リスク | 5x | 資金の2% | 経験者（1年以上） |
| ハイリスク | 10x | 資金の3% | 上級者のみ |

#### 計算機が考慮していないコスト

| コスト項目 | 概算値 | 影響 |
|-----------|--------|------|
| **Taker手数料** | 片道0.04%、往復0.08% | 10万円ポジションで80円/トレード |
| **Maker手数料** | 片道0.02%、往復0.04% | 指値エントリーなら半額 |
| **スリッページ** | 0.01〜0.05% | 成行注文時。大きなポジションほど悪化 |
| **Funding Rate** | 8時間ごと | 0.01%/8h = 年率10.95%。保有時間が長いと無視できない |

**実質コスト例**:
```
ポジション: 100万円分のBTC
レバレッジ: 5x（証拠金20万円）
往復手数料: 100万 × 0.08% = 800円
スリッページ: 100万 × 0.02% = 200円
合計固定コスト: 約1,000円

→ 1回のトレードで最低1,000円のプラスが出ないと赤字
→ SLまでの距離が0.5%（5,000円の損失）の場合、
   実質損失 = 5,000 + 1,000 = 6,000円
```

### 6.2 1%ルールの徹底

**1トレードで失う最大額 = 総資金の1%**

これは世界中のプロトレーダーが共通して守る鉄則。

| 総資金 | 1%の最大損失 | 10連敗しても残る資金 |
|--------|------------|-------------------|
| 50万円 | 5,000円 | 約45.5万円（生存可能） |
| 100万円 | 10,000円 | 約90.4万円 |
| 500万円 | 50,000円 | 約452万円 |

**なぜ1%か**: 10連敗（確率的にあり得る）しても資金の90%以上が残り、復帰可能。3%ルールでは10連敗で資金の26%を失い、心理的に崩壊する。

### 6.3 損切り（SL）の設定方法

#### 方法1: ATRベースSL（推奨）

ダッシュボードはATR（14期間）を計算している。これを使ったSL設定：

```
LONG の場合: SL = エントリー価格 - 1.5 × ATR(14)
SHORT の場合: SL = エントリー価格 + 1.5 × ATR(14)
```

例: BTC = $87,500、5分足ATR = $150
→ LONG SL = $87,500 - $225 = $87,275
→ ATR の 1.5倍は「通常のノイズ」の外側に設定するため

#### 方法2: ストラクチャーベースSL

直近のスイング高値/安値の少し外側にSLを設定する。ダッシュボードの「価格レンジ予測」セクションのサポート/レジスタンスレベルが参考になる。

```
LONG: SL = 直近スイング安値 - (ATR × 0.3)
SHORT: SL = 直近スイング高値 + (ATR × 0.3)
```

#### 絶対にやってはいけないこと

- **SLを遠くに動かす**（損失拡大の一方）
- **SLなしでエントリー**（一発退場のリスク）
- **「もう少しで戻るはず」と思ってSLをキャンセル**（最も危険な思考）

### 6.4 利確（TP）の段階的プロトコル

ダッシュボードのTP1/TP2/TP3に対応した利確戦略：

```
TP1（保守的）: ATR × 0.4 の利益で 40% を決済
  → SLをブレイクイーブンに移動

TP2（標準）: ATR × 0.8 の利益で 30% を決済
  → SLをTP1レベルに移動（最低利益確保）

TP3（ランナー）: ATR × 1.4 の利益で残り 30% を決済
  → または2本前のローソク足安値/高値でトレーリング
```

**例**: $87,500でLONG、ATR=$150
- TP1: $87,560（+$60）→ ポジションの40%決済、SLを$87,500に移動
- TP2: $87,620（+$120）→ 30%決済、SLを$87,560に移動
- TP3: $87,710（+$210）→ 残り30%決済

### 6.5 日次/連敗リミット

| ルール | 内容 | 理由 |
|--------|------|------|
| **1日最大損失 -3%** | 総資金の3%を失ったらその日のトレード終了 | メンタル崩壊→リベンジトレード→大損の連鎖を防ぐ |
| **3連敗で30分休憩** | 3連続で負けたら最低30分間トレードしない | 冷静さを取り戻す時間 |
| **5連敗でその日終了** | 5連敗したらPC/スマホを閉じる | 勝てない日は何をやっても勝てない |
| **週次最大損失 -8%** | 週の累計で8%を失ったらその週は終了 | 長期生存が最優先 |


### 7.3 認知バイアスの管理

スキャルピングで最も多い敗因は**技術不足ではなく心理的ミス**。以下のバイアスを認識し、意識的に対処すること。

#### リベンジトレード

**症状**: 負けた直後に「取り返す」ためにすぐエントリー。ポジションサイズを大きくしがち。

**対処法**:
- 負けた後は最低5分間チャートから目を離す
- 次のトレードは**前回より小さいサイズ**で入る
- 「取り返す」思考が浮かんだら、その日のトレードを終了する

#### FOMO（Fear Of Missing Out）

**症状**: 「もう動き始めてしまった、乗り遅れる！」と思って追いかけエントリー。すでにATRの半分以上動いた後に入ることが多い。

**対処法**:
- 「乗り遅れたトレード」は**次の押し目/戻りを待つ**
- ダッシュボードのエントリータイミングが「GO」でなければ入らない
- 「このチャンスを逃しても次がある」と自分に言い聞かせる

#### アンカリング

**症状**: 以前の利益レベルに固執する。「前回は$88,000でロングして儲かったから今回も」。

**対処法**:
- 過去のエントリー価格は無関係。毎回ゼロから判断する
- ダッシュボードのスコアだけを判断基準にする

#### 確証バイアス

**症状**: 自分が「ロングしたい」と思っているため、ロングを支持するインジケータだけを見て、ショートシグナルを無視する。

**対処法**:
- エントリー前に「自分の逆ポジションの根拠」を3つ探す
- 逆の根拠が3つ見つかったら、そのトレードは見送り
- ダッシュボードのコンフリクトパネルを必ず確認する

#### サンクコスト（埋没費用）

**症状**: 含み損のポジションを「ここまで耐えたのだから」と手放せない。SLをどんどん遠くにずらす。

**対処法**:
- **SLは一度設定したら絶対に遠くに動かさない**
- 含み損が一定額を超えたら機械的にカット
- 「エントリー時点の自分」と「今の自分」は別人。今の情報で判断し直す

### 7.4 マーケットレジーム識別

市場は常に3つのレジーム（状態）のいずれかにある。レジームごとに適切な戦略は全く異なる。

#### トレンド相場

**特徴**:
- SMA20 > SMA50（上昇）or SMA20 < SMA50（下降）が持続
- ボリンジャーバンドが拡大中
- ダッシュボードのスコアが 65以上 or 35以下 で安定
- コンフリクトが少ない（多くのTFが同方向）

**戦略**: **トレンドフォロー（順張り）のみ**。押し目買い（上昇トレンド）/ 戻り売り（下降トレンド）。逆張りは厳禁。

#### レンジ相場

**特徴**:
- スコアが40〜60で振動
- ボリンジャーバンドが収縮（スクイーズ）
- 価格がSMA20とSMA50の間で推移
- 上位足と下位足でシグナルが混在

**戦略**: **平均回帰（逆張り）**。バンド上限でショート、下限でロング。利確幅は小さく、SLはバンド外。

#### チョッピー相場

**特徴**:
- スコアが数分で LONG→SHORT→LONG と激しく変動
- タイムフレーム間のコンフリクトが多い
- ニュースや指標発表の前後

**戦略**: **トレードしない**。チョッピー相場で利益を出し続けることはプロでも困難。資金を守ることが最善の戦略。

### 7.5 「ダッシュボードがGOと言っているが入るべきでない」場面

1. **重要経済指標の発表30分前**: シグナルがどんなに良くても、指標で全てが覆る可能性がある
2. **スプレッドが通常の3倍以上に拡大**: 流動性が枯渇しており、不利約定のリスクが高い
3. **全インジケータが完全一致**: 全員が同じ方向を向いている時は「罠」の可能性。クジラが反対方向に仕掛ける
4. **アジアセッションの低ボリューム時**: 出来高が通常の30%以下なら、シグナルの信頼度が大幅に低下
5. **3連敗直後**: 心理状態が正常でない。チェックリストの7番に該当
6. **FR精算前15分**: ポジション調整のノイズが多い
7. **週末（土日）**: 流動性が極端に低下。スプレッド拡大。急な動きに対応できるトレーダーのみ

## 8. 9つの戦略の実戦運用

### ダッシュボード搭載の6戦略 + 追加3戦略

### 8.1 FR Contrarian（FRコントラリアン）— 勝率68〜74%

**概要**: Funding Rateが極端な値に達した際に逆張り

#### ダッシュボードでの具体的な見方

1. インジケータカードの「FR」が赤（▼ FR高）を確認
2. 複合スコアが 40以下（FRの影響で bearish 寄り）
3. L/S比率も同方向（ロング過多ならショート）

#### 具体的なトレード例

```
状況: FR = +0.08%/8h（異常に高い）
     L/S = 68%L（ロング過多）
     OI = 前日比+5%（新規ロングが増えている）
     ダッシュボード: 複合スコア 32（SHORT）

エントリー: ショート @ $87,500
SL: $88,375（+1%）
TP1: $87,150（-0.4%）→ 40%決済
TP2: $86,800（-0.8%）→ 30%決済
TP3: $86,375（-1.3%）→ 30%決済

根拠: FRが高すぎるためロング保有コストが大きい。
     8時間後のFR精算でロングの手じまいが起きやすい。
     L/S も一方向に偏っており、踏み上げ/清算の圧力。

時間帯: ロンドン〜NY（ボラが出やすい時間帯が最適）
サイズ: 通常の80%（逆張りのためリスクを抑える）
```

#### よくある失敗

- 強いトレンド中にFRが高止まりするケース。トレンド方向を確認せずにFRだけで逆張りすると、トレンド継続で損切りになる
- **対処**: 4H/1Dのトレンドが明確にFR方向と同じ場合は見送り

### 8.2 Liquidation Hunt（清算ハント）— 勝率63〜70%

**概要**: 清算集中帯に向かう動きにトレンドフォロー

#### 具体的なトレード例

```
状況: CoinGlassで $89,000付近にSHORT清算が大量集中
     現在価格 $87,800
     ダッシュボード: 4H LONG、複合スコア 67

エントリー: ロング @ $87,800
SL: $87,300（-0.57%、直近スイング安値の下）
TP: $88,800（清算帯の手前。清算帯を突き抜けると反転リスク）

根拠: 清算帯は価格の磁石。$89,000の清算が起きると
     強制決済のショートカバー（買い戻し）で急騰する。
     TP は清算帯の中心ではなく「手前」に設定するのがコツ。
```

### 8.3 OI Divergence（OIダイバージェンス）— 勝率65〜72%

**概要**: 価格とOIの方向が乖離したときの逆張り

#### ダッシュボードでの見方

1. インジケータマトリックスで OI の変化率を確認
2. 価格チャートとOIの方向が逆行していないか確認

```
強気ダイバージェンス:
  価格: 下落中
  OI: 上昇中（新規売りが入っている）
  → 売り手が溜まりすぎ。踏み上げ（ショートスクイーズ）の準備

弱気ダイバージェンス:
  価格: 上昇中
  OI: 急上昇（無謀なロングが積み上がっている）
  FR: +0.03%以上
  → ロング過多。清算ドミノの準備
```

### 8.4 CME Gap Fill（CMEギャップ埋め）— 勝率71〜76%

**概要**: CME先物の土日ギャップは統計的に70%以上の確率で埋まる

```
チェック手順:
1. 月曜 09:30 JST（CMEオープン前）
2. TradingView で CME:BTC1! チャートを開く
3. 金曜終値と現在価格の差を計算
4. ギャップ > 1.5% なら有効

エントリー: ギャップを埋める方向
  金曜終値 > 月曜始値 → ロング（上に埋めに行く）
  金曜終値 < 月曜始値 → ショート（下に埋めに行く）

期限: 月〜水が最も埋まりやすい。木曜まで持つ場合はスキャルではなくスイング
```

### 8.5 Cash & Carry（キャッシュ＆キャリー）— 勝率85%+

**概要**: 現物ロング + 先物ショートでデルタニュートラル。FRを受け取り続ける低リスク戦略

スキャルピングではないが、**余剰資金の運用先**として知っておくべき。

```
条件: FR が安定的に +0.01%/8h 以上
     ベーシス（先物プレミアム）が年率5%以上

実行:
1. 現物BTCを購入
2. 同額の先物ショートを建てる
3. 価格リスクはゼロ（現物の損益と先物の損益が相殺）
4. FRの受け取りが純利益

月利: FR平均0.03%/8h の場合 → 約2.7%/月 → 年率32%

リスク: FRがマイナスに転じた場合はコスト発生
       → FR が -0.01% を2連続で記録したらクローズ
```

### 8.6 Fear & Greed Reversal（F&G逆張り）— 勝率62〜70%

**概要**: F&G指数の極値で逆張り。日足レベルの大局判断

```
F&G ≤ 20: 極度の恐怖 → 大衆がパニック売り → ロング候補
F&G ≥ 80: 極度の強欲 → 大衆が楽観 → ショート候補

スキャルでの使い方:
- F&G極値の日は、ダッシュボードの複合シグナルと同方向なら確信度を上げる
- F&G極値 + FR極値が同方向なら最高精度
```

### 8.7 【追加】VWAP バウンス — 勝率60〜68%

**ダッシュボードに搭載されているが戦略としては未定義の手法**

```
条件:
- 1HのSMAトレンドが明確（LONG or SHORT）
- 価格がVWAPに接近（VWAP近傍の表示）
- エントリータイミングが WAIT_ENTRY → GO に変わるタイミング

ロングの場合:
- 上昇トレンド中に価格がVWAPまで下落
- VWAPで反発を確認（1分足でハンマー等）
- VWAP + ATR×0.1 上でエントリー
- SL: VWAP - ATR×0.5

注意: 現在のVWAPはセッションリセットがないため、
     長時間のセッションでは精度が低下する。
     TradingViewのセッションVWAPで補完推奨。
```

### 8.8 【追加】BB スクイーズブレイクアウト — 勝率65〜73%

**ダッシュボードがスクイーズを検出するが、ブレイク方向のガイダンスがない手法**

```
条件:
- ダッシュボードの BB で「⚡ スクイーズ」が表示（バンド幅 < 2%）
- スクイーズ状態が3本以上（15分足なら45分以上）継続

エントリー:
- スクイーズ後、最初にバンドの外でクローズした足の方向にエントリー
- 上バンドブレイク → ロング
- 下バンドブレイク → ショート

SL: 反対側のバンド（スクイーズ中はバンドが狭い = リスクが小さい）
TP: ATR × 1.5（スクイーズ後は大きく動くため）

確認: 上位足のトレンドとブレイク方向が一致していれば信頼度アップ
注意: スクイーズ中にエントリーしてはいけない。必ずブレイクを待つ
```

### 8.9 【追加】OI + FR コンボ — 勝率70〜78%

**最高精度の逆張りセットアップ**

```
条件（全て同時に満たすこと）:
1. OI が直近24hで +5% 以上増加
2. FR が +0.05%/8h 以上（または -0.03% 以下）
3. L/S比率が 65%以上（または 35%以下）
4. 上記3つが全て同じ方向を示している

意味: 市場が一方向に極端に偏っている
    → 反転の確率が非常に高い

エントリー:
- 全てがロング偏り → ショート
- 全てがショート偏り → ロング
- ダッシュボードの複合スコアが追随していれば確信度最高

SL: 通常の 1.5倍（逆張りのため少し余裕を持つ）
TP: ATR × 1.0（大きな反転が期待できる）

出現頻度: 月に2〜4回程度。レアだが高勝率
```
